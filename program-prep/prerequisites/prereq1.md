- Comfort with basic mouse/trackpad navigation

**Independent Practice (Pre-Session): ~2 hours**

### Motivation
Everyone has a varying levels of familiarity with their own computer or the computers around them. This session normalizes curiosity as a learning tool or a means to explore systems which is an essential professional skill beyond simply coding. By exploring a Mac without a rigid checklist, the participants are practicing exactly what engineers do daily: investigate, experiment, and learn through exploration building critical foundational mindset for remote work and self-directed learning.

### Objectives
By the end of this session, participants will:
- Develop confidence navigating Mac settings and features without fear of mistakes
- Understand that curiosity and exploration are core engineering competencies
- Build familiarity with essential Mac tools and keyboard shortcuts
- Learn that understanding your hardware (RAM, CPU, storage) is foundational to troubleshooting and performance optimization
- Practice articulating technical discoveries to peers
- Recognize the value of asking questions when confused

### Specific Things to Learn

**Technical Skills:**
- Mac system features: Mission Control, Spotlight, Activity Monitor
- Keyboard shortcuts (Command+Tab, Command+Space, Command+Q, etc.)
- Screen recording on Mac (built-in tools)
- Basic hardware vocabulary: RAM (memory), CPU (processor), GPU (graphics processor), storage/hard drive
- Finder navigation and file organization basics (hidden files, sidebar, toolbar)
- System Preferences/Settings navigation

**Soft Skills:**
- Articulating technical discoveries clearly
- Asking clarifying questions when confused
- Learning through exploration and play
- Overcoming fear of "breaking" technology
- Peer teaching and knowledge sharing

### Independent Practice
🎗️ _**All independent practice tasks should be turned in no later than one hour ahead of the guided session's start. For example, if the guided session begins at 6pm, participant work should be turned in by 5pm.**_

#### Task 1: "Explore Your Mac Scavenger Hunt" (60–90 minutes)

**Objective:** Participants will actively explore their Mac's features and document their discoveries.

**Instructions:**

1. Find and interact with 12 specific Mac features:
    - Mission Control (swipe up with three fingers or press Control+Up)
    - Spotlight search (Command+Space)
    - Activity Monitor (find running applications and processes)
    - Keyboard shortcuts (find at least 3 different ones you didn't know)
    - Control Center (top-right menu bar)
    - Finder sidebar and toolbar customization
    - Hidden files/folders toggle in Finder
    - Screen recording tool
    - System Settings (find two settings you've never seen)
    - Dock customization
    - Force Quit application
    - Screenshot tools (Command+Shift+3, Command+Shift+4, Command+Shift+5)
2. For each feature, create a 15–30 second screen recording explaining:
    - What the feature is called
    - Where you found it
    - What it does
    - Why it might be useful
3. Compile all 12 screen recordings (or one long 2–3 minute compilation) and submit the link.

**Submission:**
Share a link to screen recorded interaction with 12 Mac features

**Free Resources:**
- **[Apple Support Official Guides](https://support.apple.com/mac)**: (search for specific features like "Mission Control" or "Spotlight")
- **YouTube Channel - "Mac Basics by Apple":** Apple's official YouTube channel has short tutorials on Mac features
- **Built-in Mac Help:** Press Help menu in any Mac app for contextual guidance
- **Screenshot & Recording Tool Help:** Built into Mac—just press Command+Shift+5 and explore the interface

**Reflection Prompt:**
- Which feature surprised you most?
- Which one was hardest to find?
- Did you discover anything beyond the 12 required items?

#### Task 2: "Curiosity Log" (20 minutes)

**Objective:** Practice intentional exploration and reflection.

**Instructions:**
1. Spend 15–20 minutes clicking through Mac settings you've never explored before. Don't worry about understanding everything—just look around.
2. In a Google Doc, write a short reflection (5–10 sentences total) addressing:
    - **5 things you discovered:** List 5 specific settings, features, or options you had no idea existed
    - **3 things you don't understand:** List 3 things you saw but didn't fully grasp
    - **2 questions for peers:** What do you want to ask classmates about their Mac experience?
3. Share the Google Doc link with program staff.

**Submission:**
Share a link to google doc with 5-10 reflection “Curiosity Log” sentences 

**Free Resources:**
- [Apple Support](https://support.apple.com/mac) for any terms you encounter and want to understand better
- Built-in Mac dictionary: Highlight any word and press Command+Control+D for a quick definition


#### Task 3: "Basic Computer Hardware Vocab" (30 minutes)

**Objective:** Understand the fundamental hardware components that power Mac computers.

**Instructions:**
1. Using the "Free Resources" section below, review and learn about the following computer hardware:
    - RAM (Random Access Memory) — what it does and why it matters
    - CPU (Processor) — the "brain" of the computer
    - GPU (Graphics Processor) — handles graphics and visual rendering
    - Storage/Hard Drive — where files live permanently
2. Take brief notes on each concept (3–5 sentences per component).
3. Open Activity Monitor (Command+Space, search "Activity Monitor") and observe:
    - Memory (RAM) usage at the top
    - CPU usage for different applications
    - How storage is used (use Finder: About This Mac > Storage)
4. Answer: "Why might an engineer care about understanding RAM, CPU, and storage?"

**Submission:**
Share 2-3 sentences about understanding hardware as a future software engineer

**Free Resources:**
- **[Crash Course Computer Science - "Computer Hardware" (YouTube)](https://www.youtube.com/watch?v=QZwneRb-zqA) (18 minutes)**: excellent visual explanation of hardware components
- **[Khan Academy - "Intro to Computer Science"](https://www.khanacademy.org/computing/ap-computer-science-principles)**: search for "Hardware" unit
- **[Laptop specs explained: A jargon-free guide to what's inside your computer (article)](https://mashable.com/article/laptop-specs-explainer)**: explains Mac specs in plain language
- **[TED-Ed - "Inside Your Computer" by Bettina Bair (YouTube)](https://youtu.be/AkFi90lZmXA?si=pSqvKHAxEec8OSwb)**: 5-minute animation
- **[freeCodeCamp - "Computer Basics for Beginners" (YouTube)](https://youtu.be/y2kg3MOk1sY?si=xsBO-8MHhxGQjlPm)**: 1 hour very beginner-friendly video course
- **[Techtonica Keyboard Shortcut Practice](https://github.com/Techtonica/keyboard-shortcuts-practice)**

**Reflection Prompt:**
Write 2–3 sentences: "How does understanding hardware help me as a future software engineer?"

**Demo 1: Efficient Window & Tab Management (10 minutes)**
As a remote worker and engineer, having many windows open and knowing how to navigate efficiently, saves time and reduces frustration
- Open multiple applications (Browser, Text Editor, Finder)
- Demo Mission Control to see all open windows
- Show Command+Tab to switch between apps
- How to split screen view (drag window to side)
- How tabs work in browser
- Open more than one file in a single app and "Command+`" (the backtick or backquote, to the left of the 1 key) to switch between the windows in the current app
  - Point out that folks familiar with a similar keyboard shortcut in Windows– Alt+tab, which switches between any window regardless of which app it belongs to– will need to use two combinations on Mac, the previously shown Command+Tab to switch apps, and Command+` to switch windows in the selected app

**Demo 2: Spotlight Search Mastery (8 minutes)**
Spotlight is a best friend for finding things fast—faster than digging through folders
- Command+Space to open Spotlight
- Search for applications, files, and settings
- How Spotlight can do calculations and conversions
- How to search for recent files

**Demo 3: Screen Recording & Sharing (10 minutes)**
Participants will use screen recording throughout the program to show program staff their errors, demonstrate your work, and collaborate with peers
- Use Command+Shift+5 to open screenshot/recording interface
- Record a 30-second demo of opening an application
- Where a recording saves
- Explain when and why screen recordings are useful (demos, bug reports, teaching peers)

**Demo 4: Activity Monitor for Troubleshooting (10 minutes)**
When your computer is slow, Activity Monitor helps you figure out why. As a developer, you'll use similar tools to debug your code
- Open Activity Monitor
- Memory and CPU tabs
- How to identify which apps are using the most resources
- Demonstrate force-quitting an unresponsive app

### Summary

1. **"Essential Mac Keyboard Shortcuts Cheat Sheet"**
    - Command+Space (Spotlight)
    - Command+Tab (App Switcher)
    - Command+Q (Quit App)
    - Command+W (Close Window)
    - Command+N (New Window)
    - Command+T (New Tab)
    - Command+, (Preferences)
    - Command+` (Window Switcher within one app)
    - Command+Shift+3 (Screenshot)
    - Command+Shift+5 (Screenshot/Recording)
    - Control+Click (Right-click alternative)
2. **"Glossary of Mac & Computer Terms"**
    - Mission Control, Spotlight, Finder, Dock, Menu Bar, Activity Monitor, System Preferences, Terminal (basic intro only)
    - RAM, CPU, GPU, Storage, Hard Drive, SSD
3. **"Troubleshooting Guide"**
    - "My Mac is running slowly" → Open Activity Monitor
    - "I can't find an application" → Use Spotlight (Command+Space)
    - "An app won't close" → Force quit using Activity Monitor
    - "I don't know how to do something" → Check Apple Support or YouTube for "[Mac feature] tutorial"
