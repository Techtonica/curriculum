# Dynamic Programming

Dynamic programming (DP) is a technique for solving problems by breaking them into overlapping subproblems and storing results so you don’t recompute the same work.

DP comes up in real products all the time: scheduling, route optimization, pricing/discount rules, text comparison, and resource allocation.

## How to Study DP Effectively

1. Identify the subproblem clearly (“what does `dp[i]` represent?”)
2. Write the recurrence (how do you build `dp[i]` from smaller answers?)
3. Decide memoization (top-down) vs tabulation (bottom-up)
4. Check time/space tradeoffs and optimize if needed
